# 平台架構設計

## 目標與非目標

- 目標：非商業性質的線上桌遊平台，用於驗證獨立開發桌遊平台的可行性。
- 目標：平台層聚焦於玩家與對局關聯管理，不包含遊戲規則引擎細節。
- 非目標：本階段不設計管理員功能。
- 非目標：本階段不設計斷線重連與私人對局治理機制。

## 系統範圍與角色

- 角色：玩家（Player）。
- 外部系統：Google OAuth（僅用於會員註冊與登入）。
- 系統邊界：
  - 平台負責大廳、對局建立與加入、身份驗證、對局狀態管理。
  - 遊戲內部規則、演算法與勝負判定屬於遊戲模組範疇，不在本文件詳述。

## C4 L1 系統情境

- 玩家透過平台前端進行大廳與對局操作。
- 平台前後端共同提供身份驗證、對局清單、對局參與與狀態更新能力。
- 平台於註冊與登入流程中與 Google OAuth 互動，完成會員身份綁定。

## C4 L2 容器

### 前端 (Frontend)

- 呈現大廳、對局等待頁、對局頁與註冊頁。
- 呼叫後端 API 完成身份驗證、對局建立與加入、會員註冊。
- 維持 WebSocket 連線以接收對局狀態更新。

### 後端 (Backend)

- 提供身份驗證相關端點（如 `/self`）與會員註冊流程。
- 提供對局建立、加入、開始、中斷、結束等平台層能力。
- 維護對局狀態與玩家參與名單，並透過 WebSocket 廣播狀態。

## 核心實體

本章節說明需要持久化的資料。

### 玩家 (Player)

```yaml
id:
  type: ULID

  type: string

  type: string | null
  unique: true
```

### 對局 (Match)

```yaml
id:
  type: ULID
name:
  type: string
  name: 名稱
gameId:
  type: ULID
  name: 遊戲類型 ID
status:
  type:
    - WAITING_FOR_PLAYERS
    - IN_PROGRESS
    - INTERRUPTED
    - COMPLETED
  name: 狀態
players:
  type:
    id:
      type: ULID
    name:
      type: string
    status:
      type:
        - ONLINE
        - OFFLINE
  name: 玩家列表
setupContext:
  type: object
  name: 遊戲設定
state:
  type: object
  name: 遊戲狀態
```

## 關鍵流程

### 遊戲大廳 UX

#### 建立對局

- 使用者進到大廳。
- 點擊建立對局按鈕，出現設定對話框或頁面。
- 使用者輸入對局名稱、選擇遊戲類型、設定遊戲選項。
- 點擊建立按鈕，進到對局等待頁面，等待其他玩家加入。

#### 加入對局

- 使用者進到大廳，看到目前的對局列表。
- 點擊想加入的對局，進到對局等待頁面。
- 如果對局還有空位，或是已是對局記載的玩家 ID，使用者可以點擊加入按鈕加入對局。
- 記錄玩家 ID、名稱到對局實體的 players 欄位。
- 連上 WebSocket，狀態記錄為 ONLINE。

#### 開始對局

- 在對局等待頁面，當玩家數量達到遊戲類型的最小要求時，開始按鈕變為可用。
- 主機玩家點擊開始按鈕，對局狀態變為 IN_PROGRESS，所有玩家進入遊戲畫面。
- 依據遊戲規則開始接受各個玩家的命令，更新對局實體的 state 欄位。

#### 中斷對局

- 在遊戲中，任何玩家都可以點擊中斷按鈕，對局狀態變為 INTERRUPTED，所有玩家回到對局等待頁面。
- 繼續對局的方式如同開始對局，主機玩家點擊繼續按鈕，對局狀態變為 IN_PROGRESS，所有玩家進入遊戲畫面。

#### 結束對局

- 在遊戲中，當達到遊戲結束條件時，對局狀態變為 COMPLETED，所有玩家回到大廳。

### 會員 UX

#### 大廳身份驗證

- 使用者進到大廳，前端呼叫 `/self` 端點，依據 cookie 確認自身身份。
- 如果不是既有玩家，則建立新的玩家實體，名稱使用流水號。
- 回傳玩家資訊給前端，前端可顯示玩家名稱與對局列表等資訊。
- 前端依據是否有 email 決定是否顯示註冊按鈕。

#### 註冊會員

- 使用者點擊註冊按鈕，跳轉到 Google 登入頁面。
- Google 登入成功後，跳轉回確認端點，確認是否是既有 email 的玩家。
- 如果是既有 email 的玩家，修改 cookie 資訊，繼續既有大廳身份驗證過程。
- 如果不是既有 email 的玩家，出現繼續註冊表單，讓使用者輸入名稱並確認 email。
- 使用者點擊完成註冊按鈕，更新玩家實體的 email 欄位，修改 cookie 資訊，繼續既有大廳身份驗證過程。

#### 登出會員

- 使用者點擊登出按鈕，清除 cookie 資訊，回到大廳身份驗證過程，建立新的大廳身份。

## 頁面架構

- 主版面
  - 頁面名稱、WebSocket 連線狀態、玩家名稱。
- 大廳 (`/`)
  - 呈現當前的對局列表。
  - 註冊與登入按鈕。
  - 建立對局按鈕。
- 特定對局 (`/matches/:id`)
  - 非進行中的對局
    - 顯示對局資訊。
    - 玩家列表。
    - 加入與離開對局按鈕。
    - 開始與繼續對局按鈕（主機玩家）。
  - 進行中的對局
    - 顯示遊戲畫面。
    - 中斷對局按鈕。
- 註冊 (`/register`)
  - 顯示註冊表單，讓使用者輸入名稱並確認 email。
