# 通訊架構設計

## 元件

- `GameRule`: 遊戲規則
  - 定義遊戲的核心邏輯，包含遊戲狀態轉移規則。
  - 應當要
  - 提供 `move(playerId, action): GameState` 的函數，讓 `GameRunner` 執行。
  - `GameRule` 應當本身就能被測試
    - 如: 在 `ooo` 狀態時， `move(playerId, action)` 會回傳 `xxx` 狀態。
- `Game`: 遊戲
  - 為特定一場遊戲的實例，包含遊戲狀態 `GameState` 與玩家資訊。
- `GameRepo`: 遊戲存放庫
  - 負責存放遊戲狀態，提供 `get(gameId): Game` 與 `save(gameId, game)`。
- `GameRunner`: 遊戲執行器
  - 接收特定玩家命令，找到對應的 GameState，實例化 GameRule，執行 move(playerId, action) 以轉移遊戲狀態。
- `PlayerTransport`: 玩家通訊服務
  - 銜接 `Elysia` 的 `WebSocket` 通訊模型的 `topic` 概念。
  - 記載當前連線階段與通訊模型。
    - 當前哪些玩家在線上。
    - 當前哪些玩家在哪個遊戲中。
  - 對 `GameRunner` 提供通訊抽象
    - `send(playerId, message)`: 向特定玩家發送訊息
    - `sendGame(gameId, message)`: 向特定遊戲的所有玩家發送訊息
    - `receive((playerId, message) => void)`: 註冊接收玩家訊息的 callback
  - 銜接 `Elysia` 的 WebSocket 連線
    - `connect(ws, auth)`: 當特定的 WebSocket 連線開啟時，註冊該連線。
    - `disconnect(ws)`: 當特定的 WebSocket 連線關閉時，取消註冊該連線。
    - `onMessage(ws, message)`: 當特定的 WebSocket 連線收到訊息時，將該訊息轉發給 `GameRunner`。
  - 提供設定玩家與遊戲關係的函數
    - `playerJoinGame(playerId, gameId)`: 設定玩家所在的遊戲
    - `playerLeaveGame(playerId, gameId)`: 取消玩家所在的遊戲
- `PlayerSocket`: 以 Elysia 實作的 WebSocketApi
  - 注入 `PlayerTransport` 並接上
- `PlayerApi`: 以 Elysia 實作的 RESTful API
  - 提供玩家註冊、登入、登出等功能
  - 登入建立 `Session`
- `PlayerRepo`: 玩家存放庫
  - 負責存放玩家資訊，提供 `get(playerId): Player` 與 `save(playerId, player)`。
- `SessionRepo`: 會話存放庫
  - 負責存放玩家的登入狀態
  - Session 結構
    - `sessionId`: 會話 ID
    - `playerId`: 玩家 ID
    - `expiresAt`: 會話過期時間

## 通訊情景

### 玩家連線

前端 ->> PlayerApi: 登入
PlayerApi -->> PlayerRepo: 驗證玩家資訊
PlayerApi -->> SessionRepo: 建立 session，回傳 session id
PlayerApi -->> 前端: 提供玩家資訊與 session id
前端 -->> PlayerSocket: 建立 WebSocket 連線，並帶上 session id
PlayerSocket -->> PlayerTransport: 註冊連線，提供 session id
PlayerTransport -->> SessionRepo: 驗證 session id，取得 player id
PlayerTransport -->> PlayerTransport: 註冊 player id 與 WebSocket 連線的對應關係
PlayerTransport -->> PlayerTransport: 按 topic 模型，將 WebSocket 訂閱有關 topic（如玩家個人 topic、遊戲 topic）
PlayerTransport -->> PlayerTransport: 按照需要，廣播玩家狀態（如在線、離線）給相關玩家

### 玩家加入遊戲

前端 ->> GameApi: 加入遊戲
GameApi -->> GameRepo: 取得遊戲狀態
GameApi -->> Game: 驗證玩家是否可以加入遊戲
GameApi -->> Game: 將玩家加入遊戲狀態
GameApi -->> GameRepo: 儲存遊戲狀態
GameApi -->> PlayerTransport: 告知玩家加入遊戲
PlayerTransport -->> PlayerTransport: 按 topic 模型，將玩家訂閱該遊戲的 topic
PlayerTransport -->> PlayerTransport: 向該遊戲的所有玩家廣播玩家加入的訊息
