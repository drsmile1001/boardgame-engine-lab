# 實作匿名註冊登入流程

## 背景

- 來源
  - `docs/arch/平台架構設計.md:125` 定義會員 UX 與歡迎頁流程
  - 本提案聚焦「匿名註冊、身份維持、登出」最小可用路徑
- 目前狀態
  - 後端 `packages/backend/src/middlewares/buildRequesterProvider.ts` 以 `player.id` 直接作為 cookie value
  - 後端 `packages/backend/src/apis/PlayerApi.ts` 以 `setRequester` 綁定身份，尚未拆分 session 實體
  - 前端 `packages/frontend/src/App.tsx` 已有基本路由守衛，但 `/welcome` 尚為占位內容
  - 前端 `packages/frontend/src/pages/layouts/MainLayout.tsx` 玩家資訊仍為示意文案
- 問題陳述
  - 目前身份與玩家資料耦合，無法完整支援 session 生命週期管理
  - 登出僅能靠覆寫 cookie 行為，缺少正式 session 撤銷流程
  - 歡迎頁與主版面 UX 尚未對齊架構文件

## 目標

- 建立匿名玩家註冊到進入大廳的完整可用流程
- 導入 session 實體，分離「玩家資料」與「登入狀態」
- 提供登出 API，確保 session 與 cookie 可被正確清除
- 前端補齊 `/welcome`、路由守衛、主版面玩家資訊與登出入口

## 設計決策

- 身份模型
  - 新增 `Session` schema
  - 欄位定義

```yaml
Session:
  id: string
  playerId: string
  createdAt: number
  expiresAt: number
```

- session 儲存
  - 本提案直接採 `EntityStoreYaml`
  - 實作形式

```ts
import { EntityStoreYaml } from "@drsmile1001/entity-store";
```

- SessionStore 持久化檔案建議為 `sessions.yaml`
- 服務啟動時 init，關機流程沿用既有 safe-exit
- 常數集中
  - 新增 `packages/backend/src/Constants.ts`
  - 集中管理

```yaml
constants:
  SESSION_COOKIE_NAME: BG_ENGINE_SESSION_ID
  SESSION_LIFETIME_SECONDS: 2592000
```

- Provider 重構
  - `buildRequesterProvider` 更名為 `buildPlayerProvider`
  - Provider 僅負責解析當前玩家，不再承擔「設定 session」職責
  - 透過 `PlayerResolver` 介面解耦 session 查詢邏輯
- API 責任邊界
  - `POST /api/register/anonymous`
    - 建立玩家
    - 若 cookie 內已有 sessionId，先嘗試刪除舊 session
    - 建立 session
    - 設定 httpOnly cookie（value=session.id）
  - `GET /api/self`
    - 由 provider 解析 `requester`
    - 無身份回傳 401
  - `POST /api/logout`
    - 刪除 session（不存在也回傳成功）
    - 清除 cookie
- 前端流程
  - `/welcome`
    - 匿名開始按鈕呼叫 `POST /api/register/anonymous`
    - 成功後導向 `/`
  - Auth Guard
    - 以 `/api/self` 判斷身份
    - 收到 401 導向 `/welcome`
  - MainLayout
    - 顯示 `currentPlayer` 名稱
    - 提供登出按鈕，成功後導向 `/welcome`

## 限制與考量

- 本階段不處理 Google OAuth 流程，按鈕僅保留占位或 disabled 狀態
- Session 採 yaml 持久化，需考量檔案競爭與 I/O 錯誤處理
- `GET /api/self` 語意調整為 401，前端守衛與測試需同步更新
- 匿名重複註冊時先刪舊 session，可避免殘留無主 session

## API 細節

- `GET /api/self`
  - 成功

```yaml
status: 200
body: Player
```

- 失敗

```yaml
status: 401
body: null
```

- `POST /api/register/anonymous`
  - 請求

```yaml
body: {}
```

- 回應

```yaml
status: 200
body: Player
set-cookie:
  name: BG_ENGINE_SESSION_ID
  httpOnly: true
  maxAge: 2592000
```

- 行為
  - 若請求夾帶既有 session cookie
    - 先刪除舊 session
    - 再建立新玩家與新 session

- `POST /api/logout`
  - 請求

```yaml
body: {}
```

- 回應

```yaml
status: 200
body:
  success: true
```

- 行為
  - 有效 session: 刪除 session，清除 cookie
  - 無效或不存在 session: 直接清除 cookie，仍回傳 200

## 流程細節

- 首次匿名進站
  - 進入 `/welcome`
  - 點擊匿名開始
  - 呼叫 `POST /api/register/anonymous`
  - 後端建立 player + session，寫入 cookie
  - 前端導向 `/`
  - Auth Guard 呼叫 `/api/self` 成功，渲染頁面

- 已有舊 session 再次匿名註冊
  - 呼叫 `POST /api/register/anonymous`
  - 後端刪除舊 session
  - 建立新 player + 新 session
  - 回傳新 cookie

- 登出
  - 點擊登出按鈕
  - 呼叫 `POST /api/logout`
  - 後端刪除 session 或略過不存在 session
  - 後端清 cookie
  - 前端導向 `/welcome`

## 測試案例展開

- 後端 provider/resolver
  - 無 cookie -> `requester = null`
  - cookie 有值但 session 不存在 -> `requester = null`
  - session 過期 -> `requester = null`，可選擇同步刪除該 session
  - session 有效但 player 不存在 -> `requester = null`
  - session 與 player 都有效 -> `requester = player`

- 後端 API
  - `GET /api/self`
    - 未登入 -> 401
    - 已登入 -> 200 + Player
  - `POST /api/register/anonymous`
    - 首次註冊 -> 建 player + session + set-cookie
    - 重複註冊 -> 舊 session 刪除 + 新 session 建立
  - `POST /api/logout`
    - 有效 session -> session 刪除 + cookie 清除 + 200
    - 無效 session -> cookie 清除 + 200

- 前端
  - `/welcome`
    - 匿名按鈕成功後導向 `/`
    - API 失敗時顯示錯誤訊息
  - Auth Guard
    - `/api/self` 401 -> redirect `/welcome`
    - `/api/self` 200 -> 保留原路由
  - MainLayout
    - 顯示目前玩家名稱
    - 點登出後回 `/welcome`

## 驗證規劃

- 後端
  - `GET /api/self`
    - 無 cookie 或無效 session 時回傳 401
    - 有效 session 時回傳玩家資料
  - `POST /api/register/anonymous`
    - 建立玩家與 session
    - 若有舊 session，舊 session 已被刪除
    - 回應含正確 cookie 屬性（httpOnly、maxAge、expires）
  - `POST /api/logout`
    - session 存在時被移除
    - session 不存在時仍回傳 200
    - cookie 被清除
- 前端
  - 未登入或 401 時進入受保護路由會導向 `/welcome`
  - 在 `/welcome` 點擊匿名開始可進入 `/`
  - MainLayout 可顯示玩家名稱與成功登出
